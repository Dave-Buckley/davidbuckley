Build end-to-end machine learning capability through structured coursework (fast.ai, Andrew Ng's Machine Learning Specialization, Karpathy's Zero to Hero), public Kaggle and Hugging Face contributions, and a full applied machine learning project with a live demonstration.

## Tasks

- [ ] [fast.ai Practical Deep Learning Part 1 complete, homework notebooks committed](https://course.fast.ai/) (ML-01)

<details>
<summary>More info · ML-01 · fast.ai Part 1</summary>

**What to do**
Complete all lectures and homework of fast.ai's Practical Deep Learning for Coders Part 1. Commit every homework notebook to a dedicated public repository. The course uses PyTorch and takes a top-down "code first" approach — train models before learning the maths behind them.

**What counts as complete**
All lecture homework notebooks committed. One substantive personal project using techniques from the course (image classifier, text classifier, or tabular model) shipped.
</details>

- [ ] [Kaggle entry or public Kaggle notebook shipped](https://kaggle.com) (ML-02)

<details>
<summary>More info · ML-02 · Kaggle contribution</summary>

**What to do**
Either enter an active Kaggle competition and submit a scored entry, or publish a substantive Kaggle notebook analysing a public dataset. Both count. The notebook route is usually better value for learning since it rewards explanation, not just leaderboard rank.

**What counts as complete**
Public Kaggle profile with at least one entry or notebook linked from this portfolio.
</details>

- [ ] [Hugging Face Hub artefact: a Space demo or a LoRA-fine-tuned small model](https://huggingface.co/spaces) (ML-03)

<details>
<summary>More info · ML-03 · Hugging Face contribution</summary>

**What to do**
Either deploy a Gradio or Streamlit demo to Hugging Face Spaces (free tier), or fine-tune a small open model (GPT-2 small, Phi-3 mini, or similar) with LoRA on a task-specific dataset and upload the adapter to the Hub.

**What counts as complete**
Public Hugging Face profile with at least one Space or model linked from the portfolio.
</details>

- [ ] [Andrew Ng Machine Learning Specialization complete in audit mode](https://www.coursera.org/specializations/machine-learning-introduction) (ML-04)

<details>
<summary>More info · ML-04 · Andrew Ng ML Specialization</summary>

**What to do**
Audit the three-course Machine Learning Specialization on Coursera (free, no certificate). Covers supervised learning, advanced algorithms, and unsupervised/recommender/reinforcement learning. Theory-first complement to fast.ai's practical-first approach.

**What counts as complete**
All three courses' lectures watched, all programming assignments attempted and committed to a public repository.
</details>

- [ ] End-to-end applied ML project with full writeup and live demonstration (ML-05)

<details>
<summary>More info · ML-05 · Applied ML capstone</summary>

**What to do**
Build and deploy one end-to-end ML project: data pipeline, model training, evaluation, deployment as a live demo. Publish a substantial writeup covering problem, dataset, approach, evaluation methodology, and limitations.

**What counts as complete**
Live demo reachable at a public URL, training code and notebooks in a public repository, technical writeup published on the portfolio.
</details>

- [ ] [Karpathy Zero to Hero plus from-scratch micrograd or mini-transformer](https://karpathy.ai/zero-to-hero.html) (ML-06)

<details>
<summary>More info · ML-06 · Karpathy Zero to Hero</summary>

**What to do**
Follow Andrej Karpathy's Zero to Hero lecture series and build along — micrograd (tiny autograd engine), makemore (character-level language model), and ideally the GPT-from-scratch lecture. This is the "understand the fundamentals" course — the autograd, backprop, attention maths all built from Python and numpy.

**What counts as complete**
Working micrograd or mini-GPT implementation in a public repository with commentary annotations showing understanding of each component.
</details>

- [ ] Add fifteen ML and AI community connections on LinkedIn (NET-04)

<details>
<summary>More info · NET-04 · ML and AI community</summary>

**What to do**
Add approximately fifteen connections from the ML and AI community — Kaggle peers encountered during ML-02, fast.ai cohort members, Hugging Face contributors whose work has been referenced, and practitioners publishing applied ML writeups. Anchor connection requests to ML-02, ML-03, or ML-05 artefacts. Use template 4 in `content/linkedin-templates.md` ("senior engineer whose content has been engaged with") where applicable; otherwise template 3.

**What not to worry about**
- Cold academics — this cluster is for practitioners, not researchers.
- Low acceptance rates from very senior or high-profile accounts; the connection attempt itself is the signal to the algorithm.

**What counts as complete**
Fifteen personalised connection requests sent, anchored to specific ML artefacts shipped in this phase. Outcomes tracked privately.

**Why this phase**
ML artefacts (Kaggle notebook, Hugging Face Space, applied ML project) provide shared-domain context that engineers-only references from earlier phases cannot. This is the phase where the programme first reads as an AI engineer's profile, not a web engineer's.
</details>
